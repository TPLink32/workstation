SQLite TRANSACTIONS/事务

由 易百 翻译整理   注： 由于水平有限，欢迎指正。转载请务必注明出处。
 上一篇 下一篇  

事务是一个工作单元，对数据库执行。事务是在一个逻辑顺序，无论是由用户手动时尚或某种数据库程序自动完成的工作单位或序列。

事务是指一个或多个更改数据库的传播。例如，如果您正在创建一个记录或更新记录从表中删除一条记录，那么正在执行的事务在一张表上。重要的是要控制事务以确保数据的完整性和处理数据库错误。

实际上，许多的SQLite查询成一组，执行所有这些放在一起作为一个事务处理的一部分。

事务的性质：

事务具有以下四个标准属性，缩写ACID，通常被称为：

  • 原子性: 确保工作单位内的所有操作都成功完成，否则，该事务点故障和以往的操作中止回滚到以前的状态。

  • 一致性: 确保数据库正确地改变状态后成功提交的事务。

  • 隔离性: 使事务操作相互独立的和透明的。

  • 持久性: 确保已提交事务的结果或效果在系统发生故障的情况下仍然存在。

事务控制：

使用下面的命令来控制事务：

  • BEGIN TRANSACTION: 开始事务处理。

  • COMMIT: 保存更改，或者可以使用END TRANSACTION命令。

  • ROLLBACK: 回滚的变化。

事务控制命令只用DML命令INSERT，UPDATE和DELETE。他们不能使用创建表时，或删除它们，因为这些操作在数据库中被自动提交。

BEGIN TRANSACTION命令：

事务可以启动或简单开始使用BEGIN TRANSACTION命令。此类事务常会继续下去，直到遇到下一个COMMIT或ROLLBACK命令。不过事务处理也将回滚如果数据库被关闭或如果发生错误。以下是简单的语法来启动一个事务： 

 BEGIN ; 

 or  

 BEGIN  TRANSACTION ;

COMMIT命令：

COMMIT命令是用来将更改保存到数据库中的事务调用事务命令。

COMMIT命令保存到数据库的所有事务自上次COMMIT或ROLLBACK命令。

COMMIT命令的语法如下： 

 COMMIT ; 

 or 

 END  TRANSACTION ;

ROLLBACK命令：

ROLLBACK命令是用于撤消事务尚未保存到数据库的事务命令。

ROLLBACK命令只能用于自上次发出COMMIT或ROLLBACK命令撤消事务。

ROLLBACK命令的语法如下： 

 ROLLBACK ;

例子：

考虑COMPANY表中有以下记录： 

 ID          NAME        AGE         ADDRESS     SALARY
 ----------    ----------    ----------    ----------    ---------- 
 1             Paul          32            California    20000.0 
 2             Allen         25            Texas         15000.0 
 3             Teddy         23            Norway        20000.0 
 4             Mark          25            Rich - Mond     65000.0 
 5             David         27            Texas         85000.0 
 6             Kim           22            South - Hall    45000.0 
 7             James         24            Houston       10000.0

现在，让我们开始一个事务，并从表中删除记录age = 25，最后，我们使用ROLLBACK命令撤消所有的变化。 

 sqlite >   BEGIN ; 
sqlite >  DELETE FROM COMPANY _____ AGE  =   25 ; 
sqlite >  ROLLBACK ;

如果检查COMPANY表仍然有以下记录： 

 ID          NAME        AGE         ADDRESS     SALARY
 ----------    ----------    ----------    ----------    ---------- 
 1             Paul          32            California    20000.0 
 2             Allen         25            Texas         15000.0 
 3             Teddy         23            Norway        20000.0 
 4             Mark          25            Rich - Mond     65000.0 
 5             David         27            Texas         85000.0 
 6             Kim           22            South - Hall    45000.0 
 7             James         24            Houston       10000.0

现在，让我们开始另一个事务和删除记录从表中age = 25，最后我们使用commit命令提交的所有更改。 

 sqlite >   BEGIN ; 
sqlite >  DELETE FROM COMPANY _____ AGE  =   25 ; 
sqlite >  COMMIT ;
